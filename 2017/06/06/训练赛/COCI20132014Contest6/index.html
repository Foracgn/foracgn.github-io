TaskC：Mirko和Slavko的积木

题意

$Mirko$和$Slavko$有两堆积木，还有一个万能口袋，里面有很多很多的积木（无限个）；现在他们想把两堆积木变为两堆相同的规则积木（1~$\left\lfloor\frac{N+1}{2}\right\rfloor$递减，差为1；$\left\lfloor\frac{N+1}{2}\right\rfloor$~N递增，差为1）；他们能在积木堆上拿走一个积木，也可以加上一个积木，这两种操作都需消耗一点体力；现在需知道若要达到他们的目的至少要消耗多少体力。

（右侧为最后可能成为的一种情况）

$1<=n<=3*10^5,0<=m_i<=10^{12}$

解析

一开始觉得这道题非常的复杂。

这只能说我对于拿积木的问题没有一个比较笼统的认识，也没有很好的把握状态怎么样向有理而且容易的方向转化。

然后我就发现了世界的终极

我们可以假设第$\left\lfloor\frac{N+1}{2}\right\rfloor$个积木最后是$x$个，$\left\lfloor\frac{N+1}{2}\right\rfloor$为$mid$。

那么：
$$
ans=\left| x+mid-m_1\right|+\left| x+mid-1-m_2 \right|+…+\left| x+n-mid-m_n \right|
$$
除了$x$是未知数以为，其他的全部都是常数，根据绝对值函数的特性我们可以很快的求解这道题。

两个人的积木也是类似的想法。

TaskD：神奇的麦田怪圈

题意

小C去乡下玩，发现了好多麦田怪圈，这里有$n$个圈。神奇的是居然没有两个圈相交（存在相切）。

现在已知的是每个圈的圆心$x_i$和$r_i$，每个圆不相同。

科幻迷小C想知道怪圈将田野分成了几个部分？（包括圈外的部分）

$1<=n<=3*10^5,-10^9<=x_i<=10^9,1<=r_i<=10^9$.

解析

对于这道题唯一需要的结论就是，如果一个圆内有几个小圆完全覆盖了它的直径，那么就会多分出一个部分。

也就是说，答案就是$n+1$+(能够被小圆完全覆盖的大圆的个数)。

第一种解法比较正统。

被完全覆盖，可以想成是区间的问题。小圆覆盖大圆，只需要把小的圆先放在区间上，等到放大圆的时候看看这一段区间是否已经被完全覆盖就好了。

但是区间的权值太大，所以还需要离散，离散后线段树就可以解决这个问题。

不过我没有写，我用的是另外一种办法。

考虑到虽然一个圆外面可能有很多大圆，但是只有一个比它大的最小的一个圆，并且每个圆都满足这样的性质(假设整个平面也是一个圆)。因此可以把这个圆的归属问题看成是一棵树。

如果一个点的所有儿子正好可以把自己覆盖的区间覆盖，答案就加一。

那么问题归结于如何建造出这棵树。

首先不难想到的是把所有区间排序，但是排序的依据并不能简单的按照左端点。

在我的办法里，如果一个圆包含于另外一个圆或者的确在另外一个圆的右端，则判定为小于。

那么我们发现，对于$i$点，区间$[fp,i-1]$均是$i$在树上的子孙。而$fp$则是左端点大于等于$i$的左端点的下标最小的一个点。有了这个结论，可以递归求解，复杂度$O(nlogn)$。

反思

还有一种办法，是以上解法的优化，就是利用括号匹配的思想，用单调栈模拟区间的进入和离开(比较栈顶元素和待进入元素的左右端点)。这里不赘述现在懒以后再写。

其实我提交的第一份代码已经很接近最后一种解法，只是少了把栈顶元素弹出的操作。而且写第二种解法的时候我也犯了一个很智障的错误：没有遍历完一个节点所有的儿子就直接返回。

这里可以看出考试的时候我的确是太过于急切。为什么不能够在考试的时候静下心来呢？因为我想去看学代会啊

TaskE：Little Mirko的哈希函数

题意

$Little Mirko$ 正在研究哈希函数，功能递归定义如下：

​ • f(0) = 0
​ • f(word+letter) = ((f(word)*33) xor ord(letter))%MOD (xor为异或，letter为一小写字母，ord(letter)为letter规定的值)

规定 ord(a~z)即为1~26；MOD为$2^{M}$;

例：当M=10时，MOD=$2^{10}$：

​ • f( a ) = 1
​ • f ( aa ) = 32
​ • f ( kit ) = 438

$Mirko$ 想要找出有多少个单词的长度为$N$且它的哈希值为$K$。

$1<=N<=10,6<=M<=25,0<=K<2^M$

解析

可以发现$100\%$的数据非常大，$dp$的话并不能够存下来。

但是却可以发现$N$的范围非常小。虽然$26^N$依旧没有办法。

这时候就可以应用上一种在搜索时有效降低复杂度的办法:$MeetIntheMiddle$。

不过用这种办法的时候需要$O(1)$(或者是复杂度很低的办法)的组合其前一半和后一半。在这里的话每一次加入一个字母都需要乘以33和异或，怎么解决呢？

发现：
$$
(…(A33) xor O_{mid+1})…)=K
$$
只需要每次都把运算(异或有可逆性，乘只需要变成逆元)倒一下：
$$
A=(…(K33^{-1} xor O_n)…)
$$
这样就可以解了。

但是这里还需要解释一下33的逆元。

正常情况下，求逆元的方式都是用费马小定理来解，但是费马小定理的前提在这里不适用($gcd(x,p)==1$且$p$为质数)。

因此，我们可以采取拓展欧几里得算法来求逆元或者是欧拉定理来求。

这里仅解释欧拉定理。

根据欧拉定理：
$$
if( gcd(x,p)==1) \ then \ x^{\varphi(p) } \%p =1
$$
费马小定理适用的原因是因为质数的$\varphi$正好等于$p-1$。

在这道题里，33和所有的$2^M$互质，满足第二个条件，同时，$2^n$的$\varphi$正好为$2^{n-1}$(这里$n>1$)。

那么，33的逆元就是$33^{2^{M-1}-1}$(其实$33^{2^M-1}$也行)。
