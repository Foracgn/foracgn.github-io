APIO#4--5.7组队赛

发表于 2017-06-06

前言

怎么说呢，真是一场非常突然的考试啊，没有一点点防备。还是一场组队赛(不过我觉得应该幸亏是组队赛我一个人多半是要死的)，老队友$Spylft$意外挂机，情急之下和$nanhan$组了队，不过$nanhan$也是大佬，虽然过程经历了不少波折，但是结果还是好的，总的来说是被学长和大佬压着，排名第7(其实要不是罚时太高还能上去一点的话说每次组队赛罚时无数是不是我中诅咒了)。

Task A

题意

给出$n$个元素$A_i$，以及$k$和$x$，每一次将$n$个元素按照大小排序，对于第奇数个元素我们将它的权值异或$x$，$k$次操作后，输出其最大的元素和最小的元素。

$1<=n<=10^5,0<=k<=10^5,0<=x,A_i<=1000$.

解析

不难发现每个元素权值不大，因此采取计数排序，每次操作复杂度$O(x)$，总共$k$次操作复杂度总共$O(x*k)$，所以暴力可过。

但是这里有个细节需要注意，就是$x$的实际范围是$[0,1023]$，因为$512\ xor\ 511=1025$.

Task B

题意

有$n$个需要的龙晶，每一天每个龙晶随机出现，其概率相同的(均为$\frac{1}{n}$)。

给出$q$，表示有$q$次询问，对于每次询问给出$p_i$，求出拿到所有龙晶概率大于等于$\frac{p_i}{2000}$的最小天数。

解析

从理论上来说应该是不怎么难的概率$DP$，但是考试总是玄学的，而且我一看到概率的题目就懵逼了啊。

唯一的结论就是，所有的龙晶没有区别。

直接给出转移方程和状态定义：

$dp[day][i]$表示此时天数一共拿到$i$个龙晶(因为龙晶没有区别)的概率。

然后进行一点点的分类讨论：

当$dp[day][n]>=\frac{p_i}{2000}$时那么$day$就是这个$p[i]$的答案。

因此，这道题也就有了很多的处理方法。第一种，发现答案满足单调结构，那么可以预处理所有的$dp[day][n]$然后二分找到答案，这里上界并不大，只有7000左右。第二种…发现$p_i$的范围不大，所以打表。第三种，为了防止超时，把所有询问排序，一遍就好了。

Task C

题意

给出$n$堆石子，每堆石子都有$A_i$个石子。

每一次可以拿走一堆石子任意多个石子，但是这一堆中已经取过的石子数不能再取，也就是说，如果第一堆中取过1个石子了，不能再在第一堆取一次1个石子，但是第二堆却可以。

两个人在玩这个游戏，无法操作的人就算输。

假使每个人都按照最优策略来玩，问后手能否获胜。

$1<=n<=10^6,1<=A_i<=60$.

解析

很明显是一个要用$SG$的博弈游戏，所以考虑怎么求$SG$了。

求$SG$值通常是用$dp$的，但是可以发现这道题的状态非常多，因为这里已经用过的步数是个非常玄学的东西，主要的问题就是存不下了。

一开始我觉得$dp$是写不出这道题的，所以就爆搜去了，我发现把所有$SG$值搜出来需要差不多三分钟。当然这并没能阻止我，机智的$Get$了需要的$SG$并不多之后我就选择打表了……

不过解决的办法还是有的，就是用$map+pair$来储存状态，用二进制来压位可用的步数。记忆化之后速度比爆搜快了大概十倍左右，可还是不够，所以还需要继续优化。

其实在爆搜前就能发现递归的层数并不会很多(否则也不会爆搜了)，那么到了后来只剩几个石子的时候就可以想到有些步数虽然还没走过，但因为石子的个数问题这些步数并没有用。去除这些步数之后(二进制去除)之后，我们就可以把许许多多的状态统一，就减少了很多的复杂度，时间大概就快了十倍，而且还是本地的速度，在$Codeforces$上提交加上输入也只用了$500MS+$，反正$AC$是够了。

时间历程:$280S->25S->2.4S$.

Task D

题意

给出$w$个桶，$f$个箱子，以及参数$h$。

要求把桶和箱子排成一排，连续的桶或箱子我们称之为一堆，一堆桶或箱子的高度定义为这一堆桶或箱子的个数。

如果所有堆的桶高度都比$h$大，那么我们把这种排列方式称为好的排列方式。

输出所有好的排列方式除以所有排列方式对$1e9+7$取模的值$(p*q^{-1} Mod 10^9+7)$。

$0<=w,f,h<=10^5$

解析

枚举箱子的堆数$i$，那么桶的堆数则为$j$，满足$|i-j|<=1$。

通过组合数可以知道箱子的排列方式有$C(f-1,i-1)$种。然后考虑桶的排列，可以知道桶一共有$C(w-1,j-1)$种排列，而合法也就是好的排列有$C(w-j*h-1,j-1)$种。

而且通过研究数据可以发现，当时，合法情况和总情况都会多一倍出来。

也就是说：

大部分的数据我们都可以用以上方法计算$p,q$来$AC$，但是这道题最玄学的就是它有一些非常鬼畜的数据。

第一种：当$f==0$时，总排列都只有一种情况，如果$w>h$的话直接输出1。

第二种：当$w==0$时，总排列也只有一种情况，但是，因为连桶都没有了，也就一定是合法的了，同样直接输出1。

反思&后记

这场考试出现了不少问题的。

首先，是有点过于害怕不熟悉的题目或者麻烦的题目了，在看到$B,C$题的时候我都是条件反射的想要交给$nanhan$来解决，但是事实上这两道题并没有很难。

然后，没有仔细看题目，思考题目也是很严重的，虽然$A$题提交这么多次部分原因是因为$nanhan$读错了题目，但是我同样在敲代码的时候没有考虑好上界的问题，而且一开始我不管怎么样都没读懂$B$题究竟想让我们做什么。

最重要的还是把握题目想要做什么，信息是怎么定义的。一道题无非就是信息的展开和信息的收束。能够把信息正确的展开就已经完成了一半。

但是不可否认，我也还是做出了某些很机智的举动的，比如说直接打表$AC$了$C$题。

这次和$nanhan$组队给了我很不一样的感觉。以前和$Spylft$组队的时候，我基本只能独立$AC$一道题，剩下就只能够和$Spylft$讨论，帮他看看代码之类的。但是和$nanhan$组队就不一样了，我感觉自己发挥了更大的作用，比如说主攻$A$题和$AC$第三题，和$nanhan$讨论第二题的优化之类的。

总之还是很愉快的。
