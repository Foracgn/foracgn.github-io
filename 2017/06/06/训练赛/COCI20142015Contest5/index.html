TaskB：贪吃蛇简化版

题意

小C想做一个贪吃蛇的游戏。在游戏中，玩家可以控制蛇在$R*S$的方格中移动，并收集所有的苹果。

但是小C能力有限，所以他制作的贪吃蛇十分低调有内涵。下面是对游戏的介绍：

1.不像传统的贪吃蛇，苹果不会逐一出现在屏幕上，而是在游戏开始时就全部展现在屏幕上。

2.在游戏的最开始，蛇的长度为1，位于屏幕最左下角的方格内，面向右方。

3.一共有两个按键操控蛇的移动，A和B。

4.按按键A，蛇会向它朝向的方向移动一个格子。如果这次移动会导致蛇走出屏幕，那么蛇就不会移动。

5.按按键B，蛇会向上移动一格，然后调头，即面对的方向转动180°。

6.如果蛇移动到了一个苹果上，那么它会吃掉苹果，并且长度不会增长。

对于一个给出的关卡，你知道最少需要按多少下按钮才可以吃掉所有的苹果吗？

$1<=R,S<=1000$

解析

模拟题很坑系列。

可以很快发现蛇在每一行的移动方向一定，想要吃到所有的苹果并且使步数最少的话只能是从每一行最左端的苹果或者最右端的苹果开始。

那么就求出每一行的最左苹果和最右苹果。这一部分的步数是一定要走的。

然后只需要模拟出蛇走的路线就可以了，也就是蛇位置的变化，需要到每一行的左右端。

TaskC：采蘑菇的摩托车

题意

小C收到了一个很酷的圣诞节礼物——一辆可以采蘑菇的摩托车。蘑菇生长在一个正方形的草原上，每一个蘑菇生长的位置可以用$(x,y)$,$(1<=x,y<=10^5)$表示。最开始草原上没有蘑菇，但是一共有$N$个蘑菇，每隔一秒都会有一个新的蘑菇在特定的空位置长出来。

机智的小C想要用他的摩托车采至少$K$个蘑菇。然而他的摩托车的形式路线与正方形草场的边或是对角线平行。由于他的摩托车跑得太快了，可以在瞬间跑完当前路径，即在下一个蘑菇长出来之前就会采完当前路径上的所有蘑菇。而且在行驶的过程中小C无法改变车的方向。

小C想只开一次摩托就采集到指定数目的蘑菇，请问他最少需要等待多少秒之后再开始采蘑菇。

没有答案输出$-1$。

$2<=K<=N<=10^6$

解析

每秒出现的蘑菇都已经知道了，只需要知道什么时候一个方向出现$k$个蘑菇就可以了。

如果只有$xy$轴就可以用数组直接记录，但是还有对角线就可能出现负数，所以想想干脆就加了一个$map$，即使慢了点也不会挂。

当然数组去一下负数也是没有什么关系的。

TaskD：帅数

题意

如果一个数的任意相邻两位奇偶性都不相同，那么我们认为这个数很帅。给出一个数 $N$ ，请问距离它最近的帅数的值。

注意：一位数都被认为很帅。两数之间的距离为两数之差的绝对值。如果有两个帅的数距离 $N$ 距离相同，那么从小到大输出两个数。

$N$最多一千位。

解析

模拟题很坑系列+1。

很容易发现这里是枚举第几位贪心后几位。

如果枚举的前几位已经不满足相邻奇偶不同的话就不成立了。

然后我们找到比这个“第几位”大的最小(个位数)和比“第几位”小的最大的(个位数)满足奇偶的两个数。

最后的几位只需要一边满足奇偶一边塞上最大${8,9}$和最小${0 ,1}$就行了。

选择答案输出的话因为高精减很麻烦，所以可以化简一下比较式($mxans+mians$和$2*str$)变成高精加就行了。

TaskE：苹果落地

题意

我们知道苹果落地的位置不会离苹果树很远。但这是真的吗？

国家统计局希望通过实践分析得到答案。他们追踪记录了 $G$ 年中某果园内所有苹果的掉落情况。果园可以看做是一个由 $R \times S$ 个方格组成的矩形。每一个方格内都可能有不止一棵的苹果树。

有趣的是，每一年只有一个苹果掉落，所以统计局一共记录了 $G$ 组坐标 $(r_i,s_i)$ ，用来表示每一年苹果掉落的位置。而且第二年，这颗苹果掉落的位置就会长出一棵新的苹果树。

我们把离它最近的那棵树当做是苹果掉落的树，你的任务就是计算每一年掉落的苹果距离离它最近的苹果树距离的平方。对于两点 $(r_1,s_1),(r_2,s_2)$ 之间距离的平方的计算公式为：

$$\Big(d\big((r_1,s_1),(r_2,s_2)\big)\Big)^2 = (r_1-r_2)^2 + (s_1-s_2)^2.$$

$1<=R,S<=500$

解析

首先考虑最暴力的解法。

对于每一次询问，都遍历整张图，找到答案最小的一个，复杂度为$O(rsG)$。

但是这里有很多明显多余的点被遍历到了。

对于遍历时的每一行(或者每一列)，可以发现只需要求出其比$s_i$大的最小的一个和比$s_i$小的最大的一个就可以了。应该可以用树状数组来求这个问题，但是这里有更加方便的解法。

考虑优化询问的过程，可以用数组$mxt[i][j]$，储存$i$行内$j$大的第一个苹果树的下标。

同样的，开一个数组$mit[i][j]$储存类似的东西。

每一次询问完，我们就把$(r_i,s_i)$记入这两个数组。

但是，如果也是全部遍历的话复杂度仍然是$O(rsG)$的。所以再加一个优化，如果遇见$mxt[i][j]>=s_i$和$mit[i][j]<=s_i$就直接返回，就好了。

为什么呢？

观察发现，这道题的图并不是很大，而如果已经被$Add$过的点在函数中会被直接$return$。

也就是说，每个点最多会被$Add$一次，每一次的复杂度最多是$O(s)$。

而这样处理之后，每次询问的复杂度最多也是$O(r)$。

那么最终的复杂度就是$O(rss+G*r)$。虽然不快，但是过这道题还是绰绰有余的(时限三秒)。

Code

在写完这道题我们发现，即使是有一些非常暴力的写法，只要做出了切合题目的判断和处理，暴力也可以没有压力$AC$。所以以后做题的时候，不妨先想想暴力是怎么写的，根据题目的设置有没有优化的余地，如果有，再算算复杂度是多少，没准会有惊喜发生因为据说暴力出奇迹。

TaskF：疯狂原始人

题意

这些年，越来越多不正常的人出现在世界上。这次的故事发生在其中十分有趣的一类人上。当然，他们就是疯狂的原始人。原始人特别多，但是只有 $N$ 个最疯狂的可以作为故事的主角，他们用 $1$ 到 $N$ 进行编号。每一个人都有一个石碑，石碑上刻着一些小写的英文单词。

有一天人猿泰山来到了疯狂原始人的聚居地，他想和他们玩一个游戏。游戏一共有 $Q$ 个回合。回合共有两种形式。以下是它们的规则：

$1$.泰山向原始人们出示单词 $P$ 。
$2$.泰山这样问其中一个原始人：“在我给你展示的所有单词中，有多少单词中包含你石碑上的单词。” （即石碑上的单词可作为多少出示单词的子串。）

因为这些疯狂的原始人已经野很久了，所以他们希望你来帮助他们完成这个游戏。告诉他们正确的答案吧。

对于每个回合：

每一行包含一个数 $O$ 。

如果 $O$ 为 $1$ ，表示第一种形式的回合，同一行包含一个字符串 $P$ ，由小写字母组成，表示泰山出示的单词。

如果 $O$ 为 $2$ ，表示第二种形式的回合，同一行包含一个整数 $S (1 \le S \le N)$ ，表示泰山询问的原始人的编号。

原始人石碑上的单词的长度之和和泰山出示的所有单词的长度之和均不超过 $2 \times 10^6$ 。

内存$1024MB$.

$1<=N,Q<=10^5$

解析

又是一道用$AC$自动机来写的题目。

话说之前那一道我还是水过去的正解还没写来着。

$AC$自动机和$Fail$树的模板就下次再谈。

那么我们考虑基于这道题是怎么写的。

可以发现，我们在自动机上遍历到的点并不能完整的概括所有的情况，也就是一个串的一个子串就是另外一个串。

那么我们就需要用到$Fail$树了。

然后，对于$Fail$上的一个节点，如果，它有一个子孙在自动机上走的时候被遍历到了，那么这个点权值也需要增加，换句话说，如果一个点被遍历了一次，那么它的所有祖先权值都需要增加，这里可以使用树状数组+$DFS$序来解决。但是可以发现，如果一个点有多个子孙都被遍历到了，那么这个点的权值就会被重复加了很多次(根据题目定义)。

怎么解决这个问题呢？

对于一个点，它如果有$c$个子孙被遍历过，就需要减去其中的$c-1$次。

而且可以发现对于每一个点都应当是这样的。

有什么办法吗？

观察每两个点，他们的$LCA$及以上就应当被减去重复的权值，同样使用树状数组+$DFS$序。

但是如果每两个点都减一遍的话肯定又会多减去很多权值。

再仔细观察之后发现，对于一个节点内所有被遍历的儿子，只需要把每两个相邻的儿子$LCA$及以上减去权值就好了。

这一部分应该是属于构造的部分，感觉空想的话是比较难想的。

这样操作完之后，一个节点(代表一个子串)及其子树权值和就是这棵树$DFS$序区间的权值的总和。

最后询问的时候，我们找到询问串的末尾是那一个节点，询问这个节点及其子树内的权值和即可。

这道题呢。同时应用了$AC$自动机+$Fail$树+树状数组+$DFS$序+$LCA$，真是一道非常好可怕的题啊。

对于模拟题的小整理

只要细心的话模拟题从来是不难的对吧？

但是对于我来说可能就有点麻烦了。

不妨总结一下怎么把模拟题写得快一点好一点吧。

首先我觉得，应该在写之前先想过五分钟，要想到大部分的可能有坑的地方，防止写代码过于简单而忽略了很多的特殊情况，特殊情况是不嫌多的。

但是即使是想到了写到了，代码不会像你想的那么好，很可能会在某些不知名的小角落里出错。因此，如果可以的话，最好还是把这部分的数据造出来。

还有就是，把代码各部分的各个功能列出来也挺好，可以方便回忆，辨别代码的正确性。相似功能的代码最好也是分开写，不要为了一点点长度牺牲代码的可读性。能够封装的也最好封装起来，比如说写成函数结构体之类的。

最后就是我的一个小发现：代码调了很长时间还是有问题的话，重写是一个比较好的选择，多写几遍对代码的理解也会加深很多。而且往往能对

据说学到最后所有题目都是模拟题所以我多半是死了。
