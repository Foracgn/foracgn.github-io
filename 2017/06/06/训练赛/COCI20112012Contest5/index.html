COCI2011/2012 Contest#5

发表于 2017-06-06

前言

非常突然的考试系列+1。

虽然事出突然心里没啥准备，不过好在题目不是很难，勉强拿了一个还不算很差的成绩。

个人觉得这套题目比较全面吧，各个方面都有涉猎，考完之后就发现有一些比较基础的算法思想应用不到位，有所警戒。

TaskA：填单词

题意

小C最近在填字游戏方面很有成就。为了在小Q面前秀智商，他决定先教小T填字游戏，拉她入坑。所以为了给身为初学者的小Q适合做的题目，小C特地准备了好多单词来制作简单版的填字游戏。

小C想请你帮忙帮他画图。小C会给你两个单词$A$和$B$，你的任务就是用这两个单词做一个简单版的填字游戏：

规定单词$A$必须水平放置，单词$B$必须竖直放置。两个单词的交叉点为单词$A$、$B$中第一个相同的字母的位置。其余的空位用“.”表示。

比如：单词$A$为“ABBA”，单词$B$为“CCBB”，那么做出的填字游戏即为：

$1<=len_A,len_B<=30$

解析

比较简单的第一题(开胃用)。

如果能够的把握住状态的话应该可以很快$AC$。

当然我是把握不住的所以用了很长时间。

TaskB：砍树

题意

小C和小Q去森林野餐，看到伐木工人在砍树。

伐木工人每次砍一行树，一行树有$n$棵，他先确定一个预设高度H，之后会把这一行树中高于$H$的部分砍下来。对于从这行树砍下来的木材，伐木工人有一个指标$M$，砍下来的木材的高度和不能小于指标$M$。小Q看了很心疼，因为伐木工人预设的高度总是太低，以至于砍了多余的木材。小Q想小C帮伐木工人预设高度，使得既能使伐木工人满足指标，又能尽可能减少被砍倒的木材的长度。你能帮帮小C么？

$1<=n<=10^6,1<=M<=2*10^9,1<=H<=10^9$

解析

一道非常基本的二分题.。

不过我感觉如果每一次$check$都把所有树都遍历一遍的话就太慢了，所以我额外处理了一个$sum$数组(先排序一遍)，然后每一次$check$都$lower_bound$出第一个比$v$大的树，直接求出了可以得到的木材长度。

但是根据实际情况来看，貌似直接循环并没有什么问题，好像还更快一点？

TaskC：DNA

题意

邪恶的生物学家找到了一段奇怪的基因片段，是由“A”，“B”组成的。他想做一个实验，前提是把所有基因片段中的“B”变成“A”，让整段基因只由“A”组成。作为他的助手，你的任务就是完成这个的实验的准备工作。现在给你两种酶，有以下功效：

每单位的酶S可以将基因片段中的单个“B”变成“A”，或将“A”变成“B”；

每单位的酶P可以将基因片段中的某一前缀片段中的“B”变成“A”，“A”变成“B”。

为了节约药品，请你算一算，你总共最少需要多少单位的酶来完成准备工作。

片段长度最长为$10^6$

解析

一开始的思路是贪心：对于一长段的$B$，我们可以直接用两次操作就使得这一段$B$全部变成$A$，而如果$B$的长度很小(其实就是单个的)的话直接变成$A$就好。

然后我就被第二组样例卡了:$BBABB$.

小部分的贪心会影响整体，于是我就悲伤地放弃了这种写法。

那么考虑$dp$好了。

操作有两种：

1、将第$i$个字母反转。

2、将前$i$个字母反转。

不难发现满足无后效性质，所以可以$dp$。

将$dp[0][i]$定义为把前面$i$个字母全部变成$A$所需要的最少操作次数，$dp[1][i]$则为把前面$i$个字母全部变成$B$所需要的最少操作次数。

最后乱递推就好了。

TaskD：填字游戏

题意

小Q对小C给她的$TOO SIMPLE$的填字游戏感到厌烦了，但是小C还是源源不断地给她编$TOO SIMPLE$的填字游戏。所以小Q想知道小C到底还有多少填字游戏可造。

小Q偷偷拿到了小C的备选单词表，一共有$n$个单词。单词表上的单词长度都相同，小C每次选择其中的4个单词，将它们首尾相连，得到一个新的填字游戏。比如，下面的填字游戏是由“HLAD”，“NIVA”，“HSIN”,“DEDA”这4个单词组成的：

其中水平放置的单词的读法为从左往右，竖直放置的单词的读法为从上到下。并且当且仅当两个单词的首或尾字母相同时，它们才可以连接。每个单词在同一个填字游戏中只能出现一次。

请你帮帮小T算一算，按照这份单词表，她总共会收到多少$TOO SIMPLE$的填字游戏（只要两个填字游戏上有一个位置上的字母不一样，他们就算不同）。

$4<=n<=10^5,2<=len<=10$

解析

首先可以得到的结论是：一个单词只有首尾两个字母是有用的。那么我们就可以直接把所有单词变成一个二维的数组。以$C[i][j]$表示首字母为$i$且末字母为$j$的单词个数。

然后枚举正方形四个端点是那几个字母，组合出这样有多少种情况。

但是很蛋疼的就是，可能有几条边用到了同一个单词，及首字母末字母是一样的情况。而减去重复情况也是比较麻烦的，如果两条边一样，那么其中一条边就要少一种选择(其实就是$A_{n}^2$)，这里我们给四条边编号为1,2,3,4.如果两条边重复则减掉编号较大的那条边的情况，这样可以保证正确。

不得不说这道题用了我非常多的时间，掌握状态的能力我一直是不足的。

所以我最讨厌组合题了。

TaskE：积木大赛

题意

XXC的幼儿园举办了一次搭积木大赛，XXC也参加了。比赛的规则如下：

1、每个参赛选手都有$N$块积木，每块积木的长度一定，都为2，高度一定都为$h$，且有一定的质量$M_i$；

2、选手只能按给出积木的顺序搭积木，不能调换积木的顺序；

3、第一块积木的左端放在起始线左边2个单位上，右端放在起始线上。之后的每次操作即把一块积木累加到上一块积木上面，但不能和上一块积木搭在同一个高度，或者更下面；

4、比赛过程中必须保证所有积木的平衡，不能倒下来。为了简化问题，积木平衡的定义是这样的：

不妨以起始线作为一条数轴的原点。

首先每块积木都有一个中心$Xcenter$，即为水平边的中点在数轴上的坐标。

其次每团积木（1块或多块积木）都有一个重心$X-barycenter$，如下计算：

如果一团积木是平衡的，那么对于其中的每一块积木，都有其上方所有积木的$X-barycenter$与它的$Xcenter$的差不超过1。这样才不会掉下来。

如上图所示，左图的摆放方案是不平衡的，因为上面两块积木的重心与第三块的中心的距离超过了1，而右图的方案是平衡的。

5、比赛胜利的规则为，在搭完这$N$块积木后，整团积木的最右端超过起始线最多的选手获胜。

XXC想请你帮忙，他怎么样才能用他的积木搭出离起始线最远的距离。

$1<=n<=3*10^5,m<=10^4$

解析

非常复杂的一道构造(或者说贪心)题。

因为很复杂所以我们还是简单来讲好了。

假如我们正着来构造的话，因为底座的位置是不变的，我们就需要非常复杂的讨论重心的问题，因此考虑倒着来构造，即每一次把积木放在最下面，那么上面积木的重心是多少呢？假设我们已经放置到了倒数第二个，现在要把最后的积木插入，最后的一个积木只能放在$[-2,0]$，那么为了使答案最大，肯定会把上面的一堆积木的重心放在$x=0$的位置上。由这种情况推广，把每一块都当做是最后一块，那么之上的一堆的重心明显落在$0$上。

然后可以发现，对于最新的一块积木，之前一堆积木的重心需要落在这块积木上，否则就塌了(貌似根据生活常识来看就是这样的)。

再考虑放置的意义，一个，就是我们把这整块积木的重心往回拉，一个，就是使整个的重心尽量向右。操作的时候我们自然是要用最少的积木达到最大的效果。

也就是说，一个新的积木，我们只有两个地方可以放上面的一堆积木，最左边和最右边。

整理以上的讨论：

首先倒着放，算出重心，算出偏移值，和之前最大的偏移值取最大。

就是这样了。

据说最后的形状一个是像一个大于号的形状，可以考虑终态来写，不过我是没怎么想过的。

事实上是考试的时候我都没时间看这道题。

TaskF：铺路

题意

现在L国需要铺设一条道路.这条道路是一条长度为$n$的字符串，字符就表示道路的花纹。
现在有$m$种瓦片,每种瓦片都有一种花纹.第$i$种瓦片,由长度为$L_i$的字符串表示.
瓦片不能翻转或者破碎.
第$i$种瓦片能够覆盖路上的某段位置当且仅当第$i$中瓦片的花纹和路上在这一段的花纹是一模一样的.
瓦片之间可以重叠,一种瓦片可以用无数次.
现在请你求出这条路有多少个位置 不能够被任何瓦片覆盖.

$1<=n<=3*10^5,1<=m,l_i<=5000$

解析

这道题倒是真的鬼畜了。

题意倒是很简单，但是问题是解法。

因为正解是自动机。

考试的时候想想貌似$KMP$的话可以写出一种$O(n*m)$的写法。

然后我就很蛋疼的开始写$KMP$了，不过因为$KMP$并不是很熟练，所以用了很久的时间，而且还错了很多的小细节。

老实说考试的时候的确要认真思考啊。

之所以这么说是因为：这道题可以用$Hash$啊。

字符串最经典的算法是什么？是$Hash$。

而且而且，$Hash$好优化……

首先对于一段符合情况的区间($Hash$值相同)，可以应用差分的写法处理好这个区间。

这样的写法是有84分的。

然后说说优化吧，我们可以先把所有的”瓦片”按照长度排序，然后从最长的串开始匹配，如果已经有一个可以匹配了，直接退出循环。因为长度越长越优。

第二个优化(其实是胡搞)，我们可以把匹配串按照尾字母分类，然后匹配时专门进入那一类串去匹配，这样就可以减少很多需要匹配的复杂度。至于为什么会用尾字母呢？因为首字母用了之后被卡了啊……

虽然虽然，的确第二个优化就是胡搞，但是写得好的话却可以逼近正解，也就是说，这是胡搞的胜利啊。
